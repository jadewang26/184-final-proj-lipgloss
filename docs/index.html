Part 1: Ray Generation and Scene Intersection

In the rendering pipeline, ray generation is where we are trying to map 2D image coordinates to 3D world-space rays. First, in our raytrace_pixel method, we sampled each pixel based on the number of camera rays in one pixel in order to take multiple samples per pixel. In order to achieve anti-aliasing, we also added a random 2D jitter using gridSampler to its pixel coordinates. Finally, we normalized these coordinates and project them onto the virtual camera sensor plane to create a vector from the camera origin to the points and apply the camera-to-world rotation matrix to them. This generates our world-space ray with min_t and max_t bounds which can be passed into the global illumination estimator to accumulate and average the radiance for the final pixel color.

For the primitive intersection part of the rendering pipeline, we tested the ray against primitive geometric shapes such as spheres or triangles to find the closest valid hit. For spheres, we used an analytical intersection test, specifically the quadratic formula, to solve for the exact hit time t. When a ray hits any primitive within its valid time bounds, we updated the ray’s max_t to the new hit time (prevents rendering obscured objects) and we populated a new Intersection object with the hit time, surface normal, and material properties to be used in the shading step of the pipeline.

In terms of implementing primitive intersection, we definitely thought the triangle intersection was much more complicated and difficult. We chose to implement the Möller–Trumbore intersection algorithm for our triangle intersection algorithm, which solves a system of linear equations to find the ray’s intersection time t and its barycentric coordinates (u, v). We extracted these values along with a third coordinate w calculated with 1 - u - v by converting the negative ray direction and triangle’s edge vectors into a 3x3 matrix and multiplying its inverse by a vector that is basically the ray’s origin to its first vertex. The intersection is considered a valid hit if t falls strictly between min_t and max_t and all three barycentric coordinates are between 0 and 1, meaning the hit point is inside the triangle. If it’s a valid hit, we would use these barycentric coordinates to interpolate the three vertex normals, allowing us to find a smooth surface normal at the point of intersection.

Part 2: Bounding Volume Hierarchy

In our construct_bvh algorithm, we first iterate through all the given primitives to create a collective bounding box. For the base case, if the total number of primitives in the current node is less than or equal to the parameter max_leaf_size, we create and return a leaf node that contains these primitives. For our recursive case, or when the number of primitives is greater than max_leaf_size, we determine the longest axis (x, y, or z) in the bounding box and use this axis as the split axis. For the splitting heuristic, we chose to calculate the average of the centroids of all primitives within the node along this split axis and partitioned said primitives into two vectors, left_partition and right_partition. This was determined based on whether the individual centroid is greater than or less than the average. Note: we avoided infinite recursion where all primitives are on one side of the split axis by forcefully popping one primitive from the full size and moving it to the empty side. Then we recursively call construct_bvh to build the left and right child nodes.

Part 3: Direct Illumination

We implemented two methods for the direct lighting function; one used uniform hemisphere sampling and the other used lighting sampling, or importance sampling:

For uniform hemisphere sampling, we added to function estimate_direct_lighting_hemisphere. First, we made a loop that iterates num_samples times (number of lights multiplied by ns_area_light). In order to get the cosine_coefficient needed for the reflection equation, inside the loop we used hemisphereSampler->get_sample() to get an incident_direction based on the sample’s orientation that we could then turn into a world_space_direction which could then be used to calculate the cosine_coefficient by taking the dot product of world_space_direction and the surface normal. To get the emitted_light and material_response for the reflection equation, we constructed a new ray that originates from hit_p with a min_t that is offset by EPS_F to prevent self-intersection. If the new ray intersects a primitive in the scene, we could get the emitted_light from the intersected surface BSDF and the material_response from the current surface BSDF, otherwise, we continue to the next sample. If it does intersect, these values can be plugged into the reflection equation: material_response * emitted_light * cosine_coefficient / (1.0 / (2.0 * PI) and added to L_out. Our final returned value is L_out / num_samples to get the average radiance of the light.

For lighting sampling, or importance sampling, we worked on estimate_direct_lighting_importance. First, we created a loop that iterates through every light in the scene. For each light, we would use a check of is_delta_light() to determine the samples for each light. We assigned the number of samples to be 1 if it was a point light and ns_area_light if it was an area light. Next, we added a loop for each sample where we called sample_L to get the incident_light_direction, distance_to_light_source, sampling_probability, and light_radiance. We can then use this to compute the BSDF and make a new ray that originates at hit_p and is bound between EPS_F and distance_to_light_source - EPS_F to avoid false BVH intersection detections (like self-intersection). This means that when we check the BVH intersections, if it returns false the light should be visible and unoccluded. This time, our equation is the Monte Carlo estimator of the reflection equation or contribution_weight * bsdf_value * light_radiance * cosine_term / sampling_probability where our sample is not added to L_out if the cosine term is negative (means the light is behind the surface) and our contribution_weight is used to multiply point lights by ns_area_light to balance the math. When all of the results of the reflection equation are added to L_out, we divide it by ns_area_light before returning to get the final averaged lighting.

The images rendered with uniform hemisphere sampling are pretty noisy and grainy in comparison to the ones that used light sampling. Part 4: Global Illumination

To implement global illumination, est_radiance_global_illumination checks if a ray intersects our scene. If depth is 0, it just returns the emitted light zero_bounce_radiance; when depth is greater than zero (and light is accumulating), total light = zero_bounce_radiance + at_least_one_bounce_radiance, where at_least_one_bounce_radiance is a recursive function that captures light bouncing multiple times between surfaces.

In our function at_least_one_bounce_radiance, we check to see if our depth is 1 (base case). If so, we return one_bounce_radiance. If isAccumBounces is true and depth > 1, we add the direct light one_bounce_radiance to our total L_out. We then use sample_f to sample an incoming lighting direction to generate the next recursive ray and decrement the depth.

For our recursive step, we check to see if isAccumBounces is true and we also check our max_ray_depth to determine if we should trace the new ray. If we have not reached our depth limit and Russian Roulette does not terminate (with 35% probability), we continue with the ray, recursively calling the function, evaluating the reflectance equation to keep accumulating light along the path.

Part 5: Adaptive Sampling

Adaptive sampling focuses the computer's processing power on the more complex parts of the image that need to be "smoothed" of noise while spending less time working on flat or empty spots. This reduces rendering time by breaking out of the ray-tracing loop early when the noise of a pixel is below the tolerable threshold.

Our implementation of adaptive sampling modifies raytrace_pixel so it stops shooting rays into a pixel once the pixel approaches a small amount of noise. We do a 95% confidence interval check through the following steps: we convert each ray's RGB radiance to a 1D scalar using Vector3D::illum(), keep track of the sum of illuminance and squared illuminance, and calculate mean and variance of the samples. If noise is low enough, the loop breaks early. Additionally, by using a modulo check, we only run the math periodically to avoid slowing down the renderer. Our sampleCountBuffer keeps track of the number of samples taken right before the loop breaks so that we can plot the data in our heat map.
